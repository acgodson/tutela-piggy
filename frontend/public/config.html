Prerequisites

  • V380 Pro camera (already set up with V380 Pro mobile app)
  • MicroSD card (any size, FAT32 formatted)
  • Computer with SD card reader
  • SIM card with active data plan (MTN, Airtel, Glo, etc.)
  • Your Farm ID (provided by Tutela team during onboarding)
1

Create the Configuration File

On Windows:

  1. Insert MicroSD card into your computer
  2. Open Notepad
  3. Paste this content: 
    [CONST_PARAM]
rtsp_enable = 1
rtsp = 1
rtsp_ctrl = 1
  4. Click File → Save As
  5. Change "Save as type" to All Files
  6. Name it exactly: ceshi.ini
  7. Save to the root of the SD card (not in any folder)

On Mac:

  1. Insert MicroSD card
  2. Open Terminal and run:
echo "[CONST_PARAM]
rtsp_enable = 1
rtsp = 1
rtsp_ctrl = 1" > /Volumes/YOUR_SD_CARD/ceshi.ini

Replace YOUR_SD_CARD with your SD card's name.

On Linux:

echo "[CONST_PARAM]
rtsp_enable = 1
rtsp = 1
rtsp_ctrl = 1" > /media/$USER/YOUR_SD_CARD/ceshi.ini
2

Apply Configuration to Camera

  1. Power OFF the camera (unplug it)
  2. Insert the MicroSD card into the camera
  3. Power ON the camera (plug it back in)
  4. Wait 5 minutes (camera reads and applies config)
  5. Power OFF the camera again
  6. Remove the MicroSD card
  7. Delete the ceshi.ini file from the card (optional but recommended)
  8. Power ON the camera
RTSP is now enabled!
3

Insert SIM Card and Connect

  1. Insert SIM card into the camera's SIM slot
  2. Power on the camera
  3. Wait for the camera to connect to mobile network (LED indicator will stabilize)
  4. Open V380 Pro app on your phone
  5. Verify camera is online and streaming
4

Find Your Camera's IP Address

Option A: V380 Pro App

  1. Open V380 Pro app
  2. Tap your camera
  3. Go to Settings → Device Info
  4. Note the IP address (e.g., 102.89.45.123)

Option B: Camera's Built-in Page

Some V380 cameras expose a web interface. Try accessing:

http://[camera-ip]/
Note: With SIM card connections, your IP will be a public mobile IP assigned by your carrier. This IP will change periodically (see Dynamic IP section below).
5

Test the RTSP Stream

Using VLC Media Player (Recommended)

  1. Download VLC: https://www.videolan.org/
  2. Open VLC
  3. Click Media → Open Network Stream
  4. Enter URL: 
    rtsp://[CAMERA_IP]:554/live/ch00_0
  5. Click Play
If video appears, RTSP is working!

Using Python/OpenCV

import cv2

cap = cv2.VideoCapture("rtsp://[CAMERA_IP]:554/live/ch00_0")

while True:
    ret, frame = cap.read()
    if ret:
        cv2.imshow("V380 Stream", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
6

Register Camera with Tutela

Since SIM cards provide dynamic IPs that change, your camera needs to be registered with Tutela so we can track it.

Option A: Tutela Dashboard (Recommended)

  1. Log into Tutela Dashboard at https://tutela-piggy.vercel.app
  2. Go to Settings → Cameras
  3. Click Add Camera
  4. Enter:
    • Farm ID: Your unique farm identifier (e.g., FARM-LAGOS-001)
    • Pen ID: Which pen this camera monitors (e.g., PEN-A)
    • Camera Name: Friendly name (e.g., Main Pen Camera)
    • Expected Pig Count: Number of pigs in this pen
  5. Click Register
  6. You'll receive a Camera Token - save this!

Option B: API Registration

curl -X POST https://tutela-piggy.vercel.app/api/cameras/register \
  -H "Content-Type: application/json" \
  -d '{
    "farm_id": "FARM-LAGOS-001",
    "pen_id": "PEN-A",
    "camera_name": "Main Pen Camera",
    "expected_pig_count": 25
  }'

Response:

{
  "camera_id": "CAM-001-ABC123",
  "camera_token": "tk_live_xxxxxxxxxxxx",
  "ingest_url": "https://api.tutela.farm/ingest"
}
7

Connect Camera to Tutela Cloud

Since SIM-based cameras have dynamic IPs that change, we need a way for Tutela to reliably connect to your camera.

Option A: Request Static IP from MTN Business (Recommended)

Register your SIM under MTN Business/Enterprise to get a static IP:

  1. Visit an MTN Enterprise/Business Center (not regular shop)
  2. Register as a business (CAC certificate helps, but sole proprietor works)
  3. Request an "IoT SIM with Static IP" package
  4. Provide Tutela with your assigned static IP
Carrier Static IP Available? Est. Cost
MTN Business Yes - IoT/M2M packages ₦10,000-30,000/month (includes data)
Airtel Business Limited availability Contact Airtel Business directly
Glo Not typically offered N/A
Why Static IP? With a fixed IP, Tutela connects directly to your camera via RTSP. No apps, no cloud dependencies, no latency.

Option B: Manual IP Updates via Dashboard (Pilot Phase)

For the pilot program, if static IP isn't available, Tutela will alert you when connection fails and prompt you to update the IP:

Where to Find IP in V380 Pro App:
  1. Open V380 Pro app
  2. Tap your camera to view live feed
  3. Tap Settings (gear icon)
  4. Tap Device Info or Network Info
  5. Copy the IP Address (e.g., 102.89.45.123)
Update IP in Tutela Dashboard:
  1. Go to Tutela Dashboard → Settings
  2. Click Cameras
  3. Find your camera and click Edit
  4. Paste the new IP address
  5. Click Save - connection will retry immediately
How often does IP change? Typically every few days, or after camera power cycles. You'll get an alert when it happens - just update the IP and you're back online in seconds.

V380 Pro Capabilities (After Rooting)

After applying the ceshi.ini config, V380 Pro supports:

  • RTSP streaming - Standard protocol, works with any video software
  • ONVIF - Standard IP camera protocol for discovery and control

V380 Pro Limitations

Even after rooting, V380 Pro still has these limitations:

  • No DDNS support - Cannot auto-update hostname when IP changes
  • No FTP/HTTP/MQTT push - Cannot push frames to a server
  • No public API for P2P - V380 cloud is app-only, no programmatic access
What is ONVIF? A standard protocol that lets any compatible software discover and control the camera. With ONVIF enabled, tools like VLC, iSpy, or Tutela can auto-discover the camera on the network and get its stream URL automatically.

For production/mass deployment with dynamic IPs, we recommend cameras with FTP push or DDNS (see recommendations below).

Handling Dynamic IPs (SIM Networks)

Why IPs Change

Mobile carriers assign dynamic public IPs that change when:

  • Camera reboots or loses power
  • SIM reconnects to network
  • Carrier rotates IP pool (every few hours/days)
  • Network congestion triggers reassignment

Solutions for V380 Pro

Solution Works with V380 Pro? Recommendation
Static IP from MTN Business Yes Best option - direct RTSP connection
Manual IP Updates Yes Workable for pilot phase
ONVIF Discovery Yes (after enabling) Helps find camera on local network only
DDNS No V380 Pro doesn't have DDNS client
FTP Push No V380 Pro can't push to server
V380 P2P Cloud No public API App-only, can't integrate
Recommendation: For pilot phase, use manual IP updates. For production, get a static IP from MTN Business or upgrade to production-ready cameras (see footer).

Troubleshooting

Stream won't connect

Issue Solution
Connection refused Ensure camera and test device are on same network
Timeout Check if SIM has active data, verify signal strength
Black screen Try alternate URL: rtsp://IP:554/live/ch00_1
Authentication error Try: rtsp://admin:admin@IP:554/live/ch00_0

RTSP not enabled after config

Different V380 hardware versions need different settings. Try these ceshi.ini variations:

Version A:

[CONST_PARAM]
rtsp = 1

Version B:

[CONST_PARAM]
rtsp_enable = 1

Version C (try all):

[CONST_PARAM]
rtsp = 1
rtsp_enable = 1
rtsp_ctrl = 1
onvif = 1

Camera keeps going offline

  • Check SIM data balance
  • Verify signal strength (move camera if weak)
  • Ensure power supply is stable (use UPS if available)

High data usage

  • Reduce frame rate in V380 app settings
  • Use motion-triggered recording instead of continuous
  • Configure Tutela agent to sample 1 frame per second

Security Recommendations

  1. Change default camera credentials in V380 app
  2. Keep Camera Token secret - treat it like a password
  3. Use HTTPS for all API communications (Tutela enforces this)
  4. Update camera firmware when available

FTP Upload Configuration

On the camera (via CamHi app or web interface), configure FTP settings:

Step 1: Open FTP Settings

In CamHi app: Settings → Alarm Settings → FTP Settings

Step 2: Enter Tutela FTP Server Details

FTP Server: 46.51.143.72
Port: 21
Username: farm-lagos-001
Password: [unique per farm]
Upload Path: /images/
Trigger: Timer (every 5 minutes) + Motion Detection
File Format: JPEG
Resolution: 1080p (2MP)

Step 3: Enable FTP Upload

In CamHi app: Settings → Alarm Settings → tick "Save pictures to FTP server"

Step 4: Test Connection

Click Test button - camera will upload a test image to verify connection.

Pro Tip: Enable BOTH timer (5 min) AND motion triggers. Timer ensures regular analysis, motion catches unusual activity immediately.
